Shader課題（平行光源、光源の方向の変更、モデルのレンダリングへの適用まで）
●作っていて分かりづらかったところ、難しかったところ
　法線と入射光の内角や反射ベクトルなどの公式はサイトに載っていたのを参考に作成していたが、どこがどの役割を担っていて、どうしてそうなるのかを理解するのに時間がかかってしまった。
　mul()やnormalize()、saturate()のHLSLリファレンスやメソッドを利用しながら公式を作成するのが混乱した。ボールを光源として利用するためのところは先生のgitから引っ張ってきたのでUpdate内で光源の位置を表すGetLightPosと光源の位置を動かして鏡面反射と影を確認するために光源の操作を実装し理解するのが大変だった。

●作っていて、分かったこと、学んだこと
　mul()やnormalize()、saturate()のHLSLリファレンスやメソッドはどのように利用されるか。最初はよくどこで何ができているのかがわからなかったが進んでいくうちにregisterを目的ごとに分けて定義したりなど、どこでどのような役割が行われているかがわかった。

●作ってみた感想
　HLSLのピクセルシェーダーのところでオブジェクトのマテリアルの色や環境光、鏡面反射ができているかを確認するために戻り値に一つ一つ変数を入れて確かめると何ができていて何ができていないかがわかりやすかったので別のプロジェクトを組むときもそのようなことをやっていこうと思った。
